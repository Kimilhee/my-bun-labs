# 모드 가설 모음

> 4개 모드는 *가설*. 학생 인터뷰 5명 결과로 카드별로 **유지/기각/순위 조정/추가** 표시.
> 각 카드: 한 줄 가설 + 검증 질문 + 상태.

마지막 업데이트: 2026-05-03 · 상태: 인터뷰 전, 모든 카드 *미검증*

---

## 끊어읽기 (chunking) — walking skeleton 1순위 (가설)

- **가설**: 학생은 긴 문장을 한 호흡으로 못 읽어서 통으로 막힌다. 끊어읽기 표시가 있으면 즉시 의미 단위로 읽힘.
- **출력**: `{ chunks: [{ text }] }`.
- **검증 질문**:
  - "이 문장 처음 봤을 때 어디서 막혔어?"
  - 학생이 받은 인쇄물에 빨간펜 슬래시 흔적이 있나?
- **상태**: 미검증. 메타앱으로 데이터 수집 중.
- **차별화**: 사람 선생님 영역, 앱이 잘 안 함.

---

## 지문 분석 (passage analysis) — 가설

- **가설**: "이 지문 뭐라는 거야"가 가장 흔한 잡. 요지·구조 한 화면.
- **출력 후보**: `{ summary, mainIdea, structure: [paragraphRoles] }`.
- **검증 질문**:
  - "지문 받자마자 가장 먼저 알고 싶은 건?"
  - 학생이 노트에 한국어 의역을 적어놓는가?
- **상태**: 미검증.
- **위험**: ChatGPT/파파고로 이미 가능 — 우리만의 추가 가치는?

---

## 어휘 분석 (vocabulary) — 가설

- **가설**: 모르는 단어 매번 사전 검색하는 게 귀찮다. 지문 내 단어를 한 번에 정리.
- **출력 후보**: `{ words: [{ word, lemma, meaning_ko, exampleInPassage }] }`.
- **검증 질문**:
  - "마지막 지문에서 사전 몇 번 켰어?"
  - "이미 아는 단어를 빼고 싶은가? 어떻게 표시?"
- **상태**: 미검증.
- **위험**: 네이버사전 압도. 차별화는 "지문 맥락 기반 의미"가 가능한가에 달림.

---

## 문법 설명 (grammar) — 가설

- **가설**: 시험에 자주 나오는 구문(분사구문, 가정법, 도치 등)을 지문에서 콕 집어 설명.
- **출력 후보**: `{ points: [{ excerpt, pattern, explanation_ko }] }`.
- **검증 질문**:
  - "문법책 본 적 있어? 언제?"
  - "문법 설명 한국어로 받고 싶어, 영어로 받고 싶어?"
- **상태**: 미검증.
- **위험**: 출력이 길어 토큰 비용 ↑, 정확도 검증 비용 ↑.

# 추가 후보 (인터뷰에서 떠오른 것들 + 가설)

## 해석 한 줄 요약 — 후보

- **가설**: 모드 4개보다 더 흔한 잡일 수 있음. "한 줄로 말해줘".
- **출력**: `{ summary_ko }` (단순).
- **상태**: 미검증.

## 시험 포인트 — 후보 (내신 도메인 특화)

- **가설**: 내신 학생의 진짜 잡은 "이 지문에서 뭐가 시험에 나올까?". 출판사·학년별 시험 패턴.
- **출력**: `{ likelyQuestions: [{ type, focus, hint }] }`.
- **상태**: 미검증.
- **킬러 가능성**: 내신 도메인이라면 이게 1순위일 수도.

## TTS (소리내어 읽어주기) — 후보

- **가설**: 발음·끊어읽기 청각 학습. 끊어읽기 모드와 결합 가치.
- **상태**: 미검증. 브라우저 SpeechSynthesis로 구현 비용 낮음.

## 모르는 단어만 찍어 사전 — 후보

- **가설**: 학생이 진짜 자주 하는 행동. 지문 전체가 아닌 "단어 1개"가 출발점.
- **상태**: 미검증. 네이버사전과 정면 경쟁.

## 선생님 필기 재현 — 후보

- **가설**: 학생 노트 사진을 관찰하면(빨간펜·괄호·화살표) 선생님 칠판 필기를 디지털 재현하는 모드가 가능.
- **상태**: 미검증. 가장 야심찬 가설, 가장 정의 모호.

## 관심사 기반 유사지문 생성 — v0.2 후보 1순위

- **가설**: 같은 문법·구문 + 학생 관심사(축구/K-POP/게임)로 새 지문 생성. 분석이 아닌 *재생산* 학습.
- **상태**: 미검증. 본 앱 v0(끊어읽기) 통과 후 시도.
- **차별화**: LLM 시대 이전 불가능했던 경험. 진짜 moat 후보.
- **기술 위험**: 패턴 일치·어휘 난이도 통제 어려움. self-critique 단계 필요할 수 있음.
