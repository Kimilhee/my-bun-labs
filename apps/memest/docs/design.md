# memest — 성경 암송 복습 게임 설계서

> 2026-07-13 grill-me 세션에서 합의된 설계. 구현 중 바뀌는 결정은 이 문서에 반영한다.

## 1. 문제 정의

사용자는 수백 개의 성경 구절을 이미 암송했다. 문제는 본문 망각이 아니라 **인출 단서의 단절**이다:

- `장절 → 말씀 첫 구절` 연결이 절반가량 끊어져 있다.
- 첫 어절만 떠오르면 나머지 본문은 이어서 암송된다.
- 기존 암송카드 방식(처음부터 자유 회상)은 "모르겠다"의 반복이라 지속 불가능.

**목표**: 장절을 보면 첫머리가 자연스럽게 떠오르는 연결을 부담 없이, 게임처럼 회복한다.
**핵심 원칙**: 실패 상태를 구조적으로 제거한다. "틀렸다"가 아니라 "힌트를 더 썼다"만 존재한다.

## 2. 범위

- **복습 전용.** 신규 암송 학습 흐름은 범위 밖 (처음 암기와 복습은 완전히 다른 처리).
- BTT(AutoHotkey 타자연습 프로그램) 포팅은 범위 밖. BTT는 데이터 원천으로만 사용.
- 사용자 1명, 스마트폰 브라우저. 백엔드 없음.

## 3. 데이터

### 원천과 변환

- 원천: `BTT/data/card*.txt` (EUC-KR, 탭 구분, `@&` = 필드 내 줄바꿈). BTT의 AHK 파서와 동일 규칙으로 해석 (DEP는 말씀 필드에 탭이 섞여 3·4열 병합).
- 변환기: `scripts/convert-btt.ts` (`bun run scripts/convert-btt.ts`)
- 산출: `data/raw-utf8/` (UTF-8 변환 원본), `data/verses.json` (카드 495개), `data/parts.json` (파트 카탈로그 16개)

### 카드 스키마

```ts
{ id, part, midTitle, title, ref, text, textEn, note, order, starred }
```

- 카드 유일 단위 = (파트, 제목, 장절). **장절이 같아도 파트가 다르면 별개 카드** (제목이 다르고 제목까지 암송 대상).
- `starred`: 원본의 `*` 표시 (옛날에 잘 안 외워졌던 카드, 87개). 집중 복습 필터로 사용.
- `note`: 원본 `@&` 뒤에 붙어 있던 부가 메모 보존.

### 파트 구조 (16개)

| 파트 | 수 | 계층 |
|---|---|---|
| 5확신 / 8생활지침 | 5 / 8 | 소제목만 |
| 60구절 | 60 | 중제목 A~E × 12 |
| 시리즈 1~5 | 36×5 | 파트 대제목 + 중제목 3개(각 12) + 소제목 |
| DEP 1~8 | 242 | 대제목(파트) + 중제목 + 소제목. 구절 수 불규칙 |

- 시리즈 대제목: 하나님을 알아감 / 사랑 안에서 자라감 / 믿음 안에서 자라감 / 승리 안에서 행함 / 그리스도를 증거함
- DEP 대제목: 구원의 확신 / 경건의 시간 / 말씀 / 기도 / 교제 / 증거 / 그리스도의 주재권 / 세계비전 (18/26/37/32/30/55/26/18개)
- DEP 1은 카테고리가 대제목 자체, DEP 8(세계비전)은 중제목 없음.
- 13·60·180은 소제목만 암송, **DEP는 중제목+소제목까지 암송** (대제목은 단서로 제시됨).
- 문제 화면의 단서는 항상 `파트 + 장절` 조합 (예: `[DEP 4. 기도] 요 15:7`).

## 4. 학습 모델 — 인출 사다리

카드마다 독립적으로 사다리를 오른다. 아래로 갈수록 인출 노력이 크다.

| 단계 | 제시 | 인출 |
|---|---|---|
| A 노출 | 장절+제목+첫머리를 그냥 보여줌 (읽기만) | 없음 (프라이밍) |
| B 인식 | 짝짓기(장절↔첫구절 4쌍) / 4지선다 | 인식 |
| C 제목 회상 | 장절 + 제목 | 첫 구절 회상 |
| D 완전 회상 | 장절만 | 제목(DEP는 중제목까지)+첫 구절 회상 |

- D가 실제 암송과 동일한 조건 = 목표 상태. "제목 힌트 유무"가 C/D를 가르는 승급 축.
- **1차 구현은 C/D만.** A/B는 2차 (연결이 심하게 끊긴 카드의 재활 코스).

### 힌트 겹 (한 카드 안에서, 못 떠올릴수록 벗겨짐)

```text
단서 제시 → [힌트] 탭마다 한 겹씩 추가 → 마지막은 정답 공개
C: 초성(ㄷㅈ ㅎㄴㄴㄲㄹㅅ…) → 첫 어절("대저 …") → 정답
D: 제목 → 초성 → 첫 어절 → 정답
```

- 복습 시 항상 맨몸(단서만)부터 다시 시작. 힌트 겹은 제시용이 아니라 **측정용**.
- **힌트 사용량 = 그 회차의 채점 점수.** 유일한 자가신고는 [떠올랐다] 탭과, 정답 확인 후 [잘못 떠올림] 자진 신고뿐.

### 어절 열기 (더블탭)

음성인식(STT) 채점은 정확도 문제로 전면 제거됨 (경위: ADR-14 → ADR-15). 남은 보조 장치:

- 문제 화면(단서 영역)을 **두 번 터치하면 본문이 앞에서부터 한 어절씩 열린다** (열린 수 `N/M` 표시). 카드가 바뀌면 초기화.
- 사용자는 소리 내어(또는 속으로) 스스로 암송하다가 막히면 두 번 터치로 다음 어절을 확인하고, [말씀 확인]으로 전문을 본 뒤 자가 판정한다 ([다시하기] / [잘못 떠올림] / [다음]).

### 첫머리 음성 확인 (ADR-16)

- [🎤 첫머리 암송] → 첫머리(공백 제외 10글자가 걸치는 어절까지)를 말하면, **글자 단위 LCS 50%** 이상일 때 통과. 맞는 순간 자동 종료.
- 통과: "삥뽕"+⭕ 잠깐 → **전체 말씀 화면** ([다시하기]/[잘못 떠올림]/[다음] — 자동으로 다음 카드로 넘어가지 않음).
- 불통과: 일치율 표시 후 문제 화면 유지(재시도 가능).
- 힌트 3겹째 "첫머리"는 기본 2어절, 두 어절이 공백 제외 5자 이내면 3어절.

## 5. 스케줄링 — 단순 Leitner

FSRS 등 본격 SRS는 도입하지 않는다 (한 번 외운 내용의 재연결이라 근사면 충분).

- 박스 0~5, 간격(일): `[1, 2, 4, 8, 16, 32]`
- **첫 만남 = 진단 배치** (별도 진단 테스트 없음. 미진단 카드는 C 형태로 제시):
  - 힌트 0 → D, box 3 / 힌트 1 → C, box 2 / 힌트 2+ → C, box 1 / 정답 공개·오회상 → C, box 0
- **일일 세션 채점** (힌트 수 기준):
  - 0개: box+1, 간격 연장. C에서 2회 연속 0개면 D 승급
  - 1개: box 유지, 간격 절반
  - 2개+: box−1, 내일 다시
  - 정답 공개/잘못 떠올림: box 0, D→C 강등, 내일 다시
- **집중 세션은 비대칭 반영**: 약함 신호(힌트 2+, 정답 공개)만 강등·간격 단축으로 반영. 성공은 간격을 연장하지 않음 (단기기억 부풀림 방지). 장기 간격을 늘리는 건 일일 세션의 성공만.

## 6. 세션

### 일일 세션

- 크기 설정 가능 (기본 20). due 도래 카드 우선, 남는 자리에 미진단 카드 순서대로.
- 정답 공개까지 간 카드는 같은 세션 뒤쪽에 1회 재등장 (재등장 회차는 채점에 반영 안 함).

### 집중 세션 (intensive)

- 스케줄 무시. 범위(파트 다중 선택 + 별표만 필터)와 개수를 직접 지정. 200개도 가능.
- **재큐잉 완주 규칙**: 힌트를 썼거나 정답 공개된 카드는 큐 뒤로 재삽입, **장절만 보고 열릴 때까지** 반복. 세션 종료 = 범위 내 전 카드가 맨몸으로 열린 상태. (채점은 첫 회차만 반영)

### 종료 화면

"회복" 프레이밍: 오늘 연결 N개 회복 / M개 강화. 점수·정답률·스트릭 없음.

## 7. UI — 화면은 하나, 나머지는 시트

탭바·네비게이션 없음. 앱을 열면 **진행 중이던 세션 카드가 그대로 복원**된다. 진행 중 세션이 없을 때만 시작 화면(복습 대기 수 + 일일/집중 세션 시작).

```text
┌─────────────────────────┐
│ [DEP 4. 기도]  12/20  ⚙ │  파트 배지 · 진행 · 설정(풀스크린 시트)
│       요 15: 7          │  단서 (C단계면 제목 포함)
│   (힌트 겹이 여기 쌓임)   │
│─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ │
│ ≡ 지나온 구절            │  바텀시트 핸들
│ [ 힌트 ]   [ 떠올랐다 ]  │  엄지 존 큰 버튼 2개
└─────────────────────────┘
```

- [떠올랐다] → 정답 공개(제목 계층+말씀 전문) → [다음]. 오회상은 [잘못 떠올림] 보조 버튼으로 자진 신고.
- **바텀시트 = 이전 구절 확인** ("직전 구절 뭐였지?" 용례): 반 올리면 최근 지나온 카드, 끝까지 올리면 범위 전체 리스트(파트 헤더 포함).
- **스포일러 방지**: 이번 세션에서 이미 확인한 카드만 `장절+제목+첫머리`(탭하면 전문). 현재·미확인 카드는 장절만.
- 설정 시트: 일일 세션 크기, 데이터 내보내기/가져오기(JSON), 진행도 초기화.

## 8. 기술

- 모노레포 디폴트: React 19 + Vite + TS strict + Biome, `bun` 런타임. 백엔드 없음.
- 저장: localStorage (진행도·설정·진행 중 세션). 백업은 JSON 내보내기/가져오기.
- 사용: 배포 주소(https://kimilhee.github.io/my-bun-labs/)가 기본. 개발 중엔 `bun dev`(`--host`) → 폰이 같은 Wi-Fi로 접속.
- shadcn/ui 미도입 (프로토타입 단계, 화면 3개에 과함 — 필요해지면 도입).

## 9. 마일스톤

1. **1차 (지금)**: C/D 회상 흐름 + 힌트 겹 + 진단 배치 + 일일/집중 세션 + 바텀시트 리스트 + 설정/백업
2. **2차**: A 노출·B 짝짓기 (재활 코스), 연결 지도 시각화, 힌트별 통계
3. **이후**: 정적 호스팅 배포, PWA
