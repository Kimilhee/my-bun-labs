# memest PRD — 성경 암송 복습 게임

> 이 문서는 "무엇을, 누구를 위해, 왜" 만드는지의 기록. 기술 결정의 근거는 [ADR.md](ADR.md), 구현 설계는 [design.md](design.md) 참조. 이 세 문서만으로 다른 AI/개발자가 처음부터 같은 앱을 재구축할 수 있어야 한다.

## 1. 배경과 문제

사용자는 네비게이토 계열 암송 코스로 성경 구절 **495개**를 이미 암송했던 사람이다. 문제는 본문 망각이 아니라 **인출 단서의 단절**:

- `장절 → 말씀 첫 구절` 연결이 절반가량 끊어져 있다.
- 첫 단어·첫 어절만 떠오르면 나머지 본문은 이어서 암송된다.
- 기존 암송카드(장절만 보고 자유 회상) 방식은 "모르겠다 → 다음 → 또 모르겠다"의 반복이라 심리적 부담이 크고 지속 불가능하다.

**핵심 목표**: 장절을 보면 첫머리가 자연스럽게 떠오르는 연결을, 부담 없이 게임처럼 회복한다.

## 2. 사용자와 사용 환경

- 사용자 1명 (개인 도구). 주 사용 기기는 **스마트폰** (Android + iOS).
- 설치형 PWA로 사용, 오프라인에서도 복습 가능해야 함 (음성 채점만 온라인 필요).
- 데스크톱 브라우저는 보조.

## 3. 원하는 경험 (제품 원칙)

1. **실패 상태를 구조적으로 제거한다.** "틀렸다"가 아니라 "힌트를 더 썼다"만 존재. 최악의 경험도 "정답을 읽고 넘어감"이지 "실패"가 아니다.
2. **회복 프레이밍.** 점수·정답률·스트릭 없음. "오늘 연결 N개 회복" 식의 요약만.
3. **부담 최소화.** 세션 크기는 작고 설정 가능. 수백 개 전체 큐는 보이지 않는다.
4. **실제 암송으로 확인한다.** 자가신고(떠올랐다)보다 음성 암송 채점이 기본. 단, 자가 확인 모드도 선택 가능.
5. **탭 몇 번으로 끝나는 조작.** 한 손, 엄지 존.

## 4. 데이터 (복습 대상)

원천: 사용자가 과거 만든 AutoHotkey 타자연습 프로그램(BTT)의 EUC-KR 카드 파일. **파트 16개, 카드 495개**:

| 파트 | 수 | 계층 | 암송 대상 제목 |
|---|---|---|---|
| 5확신 | 5 | 소제목 | 소제목 |
| 8생활지침 | 8 | 소제목 | 소제목 |
| 60구절 | 60 | 중제목(A~E) × 12 | 소제목 |
| 시리즈 1~5 (각각 별도 파트) | 36×5 | 대제목 + 중제목 3개(각 12) + 소제목 | 소제목 |
| DEP 1~8 (각각 별도 파트) | 242 | 대제목 + 중제목 + 소제목 (수 불규칙) | **중제목+소제목** |

- 시리즈 대제목: 하나님을 알아감 / 사랑 안에서 자라감 / 믿음 안에서 자라감 / 승리 안에서 행함 / 그리스도를 증거함. 시리즈1 중제목은 "예수 그리스도"가 공식 명칭.
- DEP 대제목: 구원의 확신 / 경건의 시간 / 말씀 / 기도 / 교제 / 증거 / 그리스도의 주재권 / 세계비전 (18/26/37/32/30/55/26/18개).
- **장절이 같아도 파트가 다르면 별개 카드** (제목이 다르고, 제목까지 암송 대상). dedupe 금지.
- 원본의 `*` 표시(예전에 잘 안 외워졌던 카드 87개)는 `starred` 메타데이터로 보존 — 별표만 집중 복습 필터에 사용.
- 문제 단서는 항상 `파트 + 장절` (예: `[DEP 4. 기도] 요 15:7`).

## 5. 기능 요구사항

### 학습 메커니즘 (상세: design.md §4~5)

- 인출 사다리: A 노출 → B 인식 → **C 제목 회상 → D 완전 회상** (1차 구현은 C/D만; A/B는 재활 코스로 2차).
- C단계 힌트 겹: 초성 → 첫 어절 → 정답 공개. D는 제목부터. 힌트 사용량이 곧 채점.
- 복습 전용. 신규 암송 학습은 범위 밖 (BTT 포팅도 범위 밖).
- 단순 Leitner 스케줄링 + 첫 만남 = 진단 배치.

### 세션

- **일일 세션**: 크기 설정 가능(기본 20). due 우선 + 미진단 채움. 정답 공개 카드는 세션 내 1회 재등장.
- **집중 세션**: 범위(파트·별표만)와 개수 지정, 스케줄 무시, 못 연 카드는 맨몸으로 열릴 때까지 재큐잉("답변 못하는 구절 없이 완주"). 성공은 간격을 연장하지 않음(비대칭 반영).
- **복습 범위**: 파트 체크박스 시트(취소/적용 방식)로 일일 세션 범위 제한. 세션 화면 파트 배지와 시작 화면에서 접근.

### 음성 암송 채점 (상세: design.md §4 음성 암송 채점)

- [암송 시작] 탭 → 말하기 → [탭하여 채점]. 브라우저 내장 STT(ko-KR).
- 어절 단위 자모 LCS 채점. 일반 어절 80%, **옛말 어절 50%** 통과(옛말 사전 관리).
- 100% 완벽(일반 어절 전부 일치 + 옛말 75% 이상)일 때만 "삥뽕"+⭕ 자동 다음 — 자동넘김은 설정으로 끌 수 있음.
- 그 외에는 대조 화면: 본문·받아적기 **양방향 하이라이트**, 옛말 통과 어절은 본문 단어로 교정 표시, 실제 자모 일치율 표시. [다시 암송]/[틀렸음]/[다음|맞은 걸로].
- **첫소절 매칭 모드**(설정): 처음 10글자(공백 제외)가 걸치는 어절까지만 채점.
- 음성 미지원/음성 암송 꺼짐 → [말씀 확인] 자가 확인 모드로 폴백.

### 화면 (상세: design.md §7)

- 화면은 세션 하나. 나머지(설정·구절 리스트·범위 선택)는 전부 시트. 앱을 열면 하던 세션이 그대로 복원.
- 본문 영역만 스크롤, 하단 버튼은 위치 고정 (엄지 존, 버튼 높이 2배만큼 위로 올림).
- **지나온 구절 시트**: 현재 범위 전체 리스트. 이번 세션 확인 카드 + 이전 세션에서 다룬 카드는 첫머리 공개·전문 펼침·⟲ 다시 암송. 현재/대기 카드는 장절만(스포일러 방지).
- 정답 화면 [다시하기](정답 안 봤을 때만), 설정에 버전 표시.

### 저장·배포

- 백엔드 없음. 진행도·설정·세션은 localStorage. JSON 내보내기/가져오기 백업.
- GitHub Pages 배포(https://kimilhee.github.io/my-bun-labs/) + PWA 오프라인 프리캐시. 버전은 설정 하단에 표시되어 배포 반영 확인에 사용.

## 6. 비목표 (Non-goals)

- 신규 구절 암송 학습 흐름
- BTT(타자연습) 포팅
- 다중 사용자, 서버, 로그인
- 본격 SRS(FSRS), 통계 대시보드 (필요해지면 그때)

## 7. 성공 기준

- 세션을 "실패의 반복"이 아니라 "완주"로 끝낼 수 있다 (집중 세션은 구조적으로 완주 보장).
- 옛말 오인식("미쁘사"→"예쁘시고", "깨끗케"→"깨끗하게")이 틀림으로 처리되지 않는다.
- 끊어진 구절이 D단계(장절만 보고 제목+첫 구절 회상)에 도달하는 것이 "연결 회복 완료".
